텔넷
원격에서 리눅스에 접속하는 프로그램의 이름

SSH
텔넷처럼 원격에서 시스템접속할 수 있지만 모든 통신을 암호화함
텔넷 클라이언트와 리눅스 사이에 주고받는 데이터가 암호화되지 않음

FTP서버 설치
리눅스에서 설치하는 대표적인 FTP서버는 vsFTPD

이미 작성되어 있는 파일(msg.txt) 내용을 메일로 보내려고 한다. 제목은 
“Hi User”이고 수신자는 user_1일 때 메일을 보내는 명령

-다음과 같이 명령을 기입합니다.
mailx –s “Hi User” user_1 < msg.txt

텔넷과 SSH
텔넷을 사용하려면 텔넷 클라이언트와 텔넷 서버가 필요함
텔넷 클라이언트와 텔넷 서버가 설치되어 있는지 확인해야 함
SSH로 처음 접속 → 암호화 기법인 RSA를 사용한 인증 키를 생성할 것인지 확인 → ‘yes’를 입력하면 접속
파일 송수신
FTP 서버 설치하기
리눅스에서 사용하는 대표적인 FTP 서버는 vsFTPD
FTP 송수신하기
dir이나 ls로 파일명 확인, get, mget으로 파일을 가져오거나 put, mput 명령으로 파일을 보냄

메일 송수신
메일 서버 동작 확인하기
메일 서버가 정상적으로 동작하는지, FTP와 마찬가지로 telnet으로 확인
메일 서비스의 포트 번호는 25번. 220으로 시작하는 메시지가 출력되면 잘 동작하고 있는 것임
메일 클라이언트 사용하기
리눅스에서 사용하는 기본 메일 클라이언트는 mail이나 mailx인데 둘 다 mail.mailutils의 심벌릭 링크
malix 명령으로 메일을 보내거나 읽을 수 있음

NFS: Network File System의 약자로 네트워크를 통해 다른 시스템의 디스크를 연결
NFS 서버
NFS 서버 공유 내용 확인 명령: exportfs
NFS 서버에서 익스포팅한 내용을 관리함

sync: 클라이언트가 NFS 서버에 쓰기 작업을 하면 바로 반영합니다.(기본 값)
async: 클라이언트가 NFS 서버에 쓰기 작업을 하면 바로 반영하지 않습니다.
서버에 문제가 발생했을 때 데이터 불일치 발생이 가능합니다.

IP 주소가 10.10.10.2인 시스템을 nmap으로 
포트 스캔을 하려고 한다. 다음을 처리하기 위해서는 
nmap의 옵션을 어떻게 설정해야 하는지 적어보세요.

-운영체제 스캔
nmap -O 10.10.10.2
열려있는 UDP 포트 스캔
nmap -sU -v 10.10.10.2
시스템의 TCP 포트 스캔
nmap -sT -v 10.10.10.2
80번 포트가 열려 있는지 확인
nmap -p80 -v 10.10.10.2

정보 보안의 이해
정보 보안에는 물리적인 보안, 기술적인 보안, 관리적인 보안 등 다양한 측면이 있음
정보 보안은 정보 자산을 여러 가지 위협으로부터 보호하여 기밀성, 무결성, 
가용성을 유지하는 것임
시스템 로그
로그는 커널과 리눅스 시스템이 제공하는 여러 서비스와 응용 프로그램이 발생시키는 메시지임
로그 파일 관리: journal 기능
rsyslog 데몬: 로그 통합 서버를 관리할 수 있음

ufw와 nmap
네트워크를 통한 외부의 접속을 차단하려면 방화벽(firewall)을 사용해야 함
우분투의 방화벽 명령: ufw
nmap은 내 서버나 원격의 서버가 사용 중인 포트, 운영체제 등을 스캔하여 출력함
nmap은 네트워크 관리용으로도 사용되고 취약한 포트가 사용 중인지 확인이 가능하여 
보안용으로도 사용됨

프로그래밍 개요

프로그램은 하드웨어를 동작시켜 특정 작업을 수행하는 소프트웨어
컴퓨터 프로그램 혹은 스마트폰 앱
프로그래밍은 명령하는 작업지시서를 만드는 작업

하이레벨, 로우레벨
로우레벨 - 기계어,어셈블리어(기계중심)
하이레벨-인간중심
로우레벨 = cpu가 처리하는 명령어와 일대일로 대응
하이레벨=해당 하드웨어에 맞게 번역하는 프로그램을 이용

C언어의 특징
간결하고 효율적인 언어(빠른 실행속도)
우수한 이식성(다양한 CPU와 플랫폼의 컴파일러를 지원)
다소 배우기가 어려움(비트 포인터 메모리 할등 해제 등의 관리)
